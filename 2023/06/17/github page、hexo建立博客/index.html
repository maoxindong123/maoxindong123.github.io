迁移

换电脑后，即使把hexo文件夹复制过来也不能用，所以下面是新电脑迁移的完整过程（等于重新安装了一遍了）

  1. 按照部署hexo和github page把git等等都弄了，别忘了配置git的username。如果hexo s无法打开可能是端口4000被占用，用hexo s -p 4658就可以打开

  2. 主题可以直接找之前hexo文件夹里的themes/主题名复制过来，这样前面对主题的修改就直接移过来了

    md也全部复制过来，就差不多了

  3. 如果输入github.io可以访问，但输入自己域名不能访问，是没添加cname，可以按照添加cname，但问题是，github上设置域名是自动生成一个CNAME在根目录下，但hexo d是删除github上所有文件再上传自己生成的文件，所以每次hexo d都需要重新去github上设置域名，所以靠github生成CNAME不可行

    解决方法是这个，把1个CNAME放在source文件夹下即可

  4. 如果公式显示不正确，确实需要按照Hexo完美支持数学公式走一遍就好了，单纯安装particlex主题不够

  5. 如果代码块显示不正确（行号分离、不自动换行），就按照作者github的2.1改一下，复制这3个即可
